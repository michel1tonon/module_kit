ModuleComposerBuilder<ROUTER, INJECTOR>

Convenience widget for async bootstrap in the widget tree.

Optional. Not required to use the library.

When To Use

  • You want enabled-module loading with built-in loading and errorBuilder.
  • You want simpler UI bootstrap.

Quick Example

return ModuleComposerBuilder<RouteBase, SingleChildWidget>(
  composer: composer,
  loading: const CircularProgressIndicator(),
  builder: (context, {required injectors, required routers}) {
    final router = GoRouter(routes: routers);
    return MultiProvider(
      providers: injectors,
      child: MaterialApp.router(routerConfig: router),
    );
  },
);

Retry with errorBuilder

Use a StatefulWidget and pass a changing key to force rebuild on retry:

class _RetryBootstrapState extends State<_RetryBootstrap> {
  Object _retryKey = Object();

  void _retry() => setState(() => _retryKey = Object());

  @override
  Widget build(BuildContext context) {
    return ModuleComposerBuilder<RouteBase, SingleChildWidget>(
      key: ValueKey(_retryKey),
      composer: composer,
      loading: const CircularProgressIndicator(),
      errorBuilder: (context, error, stackTrace) => Scaffold(
        body: Center(
          child: Column(
            children: [
              Text(error.toString()),
              FilledButton(onPressed: _retry, child: Text('Retry')),
            ],
          ),
        ),
      ),
      builder: (context, {required injectors, required routers}) => ...,
    );
  }
}

See example/module_kit_example_retry for a full app.

Without ModuleComposerBuilder (Manual)

final enabled = await composer.getAllEnabledModules(context);
final injectors = composer.getInjectors(context, enabled);
final routers = composer.getRouters(context, enabled);

final router = GoRouter(routes: routers);
return MultiProvider(
  providers: injectors,
  child: MaterialApp.router(routerConfig: router),
);
