FeatureModule<ROUTER, INJECTOR>

Minimal contribution contract for a module.

Main Responsibility

  • Declare module name, routes, injectors, and enabled status.
  • Can return empty (const []) when it contributes nothing.

API

abstract class FeatureModule<ROUTER, INJECTOR> {
  String get name => runtimeType.toString();
  Iterable<ROUTER> getRouters(BuildContext context) => const [];
  Iterable<INJECTOR> getInjectors(BuildContext context) => const [];
  FutureOr<bool> isEnabled(BuildContext context) => true;
}

Practical Example / Exemplo Pratico

class CheckoutModule extends FeatureModule<RouteBase, SingleChildWidget> {
  @override
  String get name => 'checkout';

  @override
  Iterable<SingleChildWidget> getInjectors(BuildContext context) => [
    ChangeNotifierProvider(create: (_) => CheckoutController()),
  ];

  @override
  Iterable<RouteBase> getRouters(BuildContext context) => [
    GoRoute(path: '/', builder: (_, __) => const CheckoutPage()),
  ];
}

Common Uses for isEnabled

  • Feature flags and experiments.
  • Role or permission checks.
  • Environment (dev/stage/prod) and remote config.
