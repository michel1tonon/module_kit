ModuleComposer<ROUTER, INJECTOR>

Aggregates enabled modules and extracts route/injector contributions.

Important Clarification

A project can have multiple ModuleComposer instances, not just one.

Usage Patterns

  • Single global composer for the entire app.
  • One composer per domain (checkout, account, admin).
  • One composer per flow (onboarding, authenticated area, feature shell).

API Flow

  1. Add modules with `addModule` or `addAll`.
  2. Resolve enabled modules with `getAllEnabledModules`.
  3. Collect contributions with `getInjectors` and `getRouters`.
final composer = ModuleComposer<RouteBase, SingleChildWidget>()
  ..addAll([
    CheckoutModule(),
    SelectCountryModule(),
  ]);

final enabled = await composer.getAllEnabledModules(context);
final injectors = composer.getInjectors(context, enabled);
final routers = composer.getRouters(context, enabled);

Design Trade-off

  • More freedom, less forced convention.
  • Requires the app shell to complete wiring.
